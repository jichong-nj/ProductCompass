# ProductCompass - 产品销售管理系统
ProductCompass 是一个基于 Django 开发的产品销售管理项目，专注于公司产品及组件的基础数据维护，通过 Django Admin 实现便捷的增删改查操作，底层采用 SQLite 数据库存储数据。

## 项目结构
```
ProductCompass/                  # 项目根目录
├── ProductCompass/              # 项目核心配置目录
│   ├── __init__.py
│   ├── asgi.py
│   ├── settings.py              # 项目配置（已注册Products应用）
│   ├── urls.py                  # 项目路由（默认包含admin路由）
│   └── wsgi.py
├── Products/                    # 产品管理核心应用
│   ├── __init__.py
│   ├── admin.py                 # Admin后台配置（产品+组件内嵌维护）
│   ├── apps.py
│   ├── migrations/              # 数据库迁移文件（自动生成）
│   ├── models.py                # 数据模型（Product + ProductComponent）
│   ├── tests.py
│   └── views.py
├── manage.py                    # Django 命令行工具
├── .gitignore                   # Git忽略文件
└── README.md                    # 项目说明文档
```

## 核心功能
### 1. 数据模型
| 模型               | 核心字段                                                                 |
|--------------------|--------------------------------------------------------------------------|
| Product（产品）| 产品名称、产品类型（平台软件/PC终端软件/硬件/单机板软件/移动APP）、产品说明（可选） |
| ProductComponent（产品组件） | 所属产品（外键关联Product）、组件名称、组件类型（软件模块/硬件）、组件说明（可选）  |

### 2. Admin 后台特性
- 产品页面内嵌组件维护（Inline 方式），支持一键增删改查；
- 产品/组件列表页支持筛选、搜索；
- 字段分组展示，时间字段（创建/更新时间）设为只读；
- 组件默认显示1个空白表单，简化新增操作。

```bash
# 安装Django
pip install django
```

## 快速开始
### 1. 克隆/下载项目
将项目文件放到本地目录，进入项目根目录（包含`manage.py`的目录）。

### 2. 数据库迁移
```bash
# 生成迁移文件
python manage.py makemigrations

# 执行迁移（创建SQLite数据库及表）
python manage.py migrate
```

### 3. 创建超级管理员
```bash
python manage.py createsuperuser
# 按提示输入用户名、邮箱、密码（密码需满足复杂度要求）
```

### 4. 启动开发服务器
```bash
python manage.py runserver
```

### 5. 访问后台
打开浏览器，访问：`http://127.0.0.1:8000/admin/`，使用超级管理员账号登录，即可维护产品及组件数据。

## 使用说明
### 1. 产品管理
- **新增产品**：进入「产品」页面 → 点击「添加产品」→ 填写产品名称、选择产品类型、补充产品说明（可选）→ 保存；
- **维护组件**：新增/编辑产品时，页面下方可直接添加/编辑/删除该产品的组件；
- **查询/筛选**：产品列表页支持按产品类型筛选、按产品名称/说明搜索。

### 2. 组件管理
- 可通过产品页面内嵌维护，也可在「产品组件」页面单独维护；
- 组件列表页支持按组件类型、产品类型筛选，按组件名称/说明/产品名称搜索。

## 注意事项
1. SQLite 数据库文件（`db.sqlite3`）已加入`.gitignore`，请勿手动提交到版本库；
2. 开发环境与生产环境建议分离配置，敏感信息（如SECRET_KEY）请勿硬编码到`settings.py`；
3. 若需部署到生产环境，需配置静态文件收集、数据库替换（如MySQL/PostgreSQL）等；
4. 迁移文件（`migrations/`目录）需提交到版本库，确保团队成员数据库结构一致。

## 常见问题
### Q1：启动服务器提示端口被占用？
修改启动命令指定端口：
```bash
python manage.py runserver 8001  # 使用8001端口
```

### Q2：忘记超级管理员密码？
重置密码：
```bash
python manage.py changepassword 用户名
```

### Q3：新增组件时提示外键关联错误？
确保先创建对应的产品，再为该产品添加组件；组件的「所属产品」字段为必填项。
